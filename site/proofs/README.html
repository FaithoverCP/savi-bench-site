How to Read a Proof Manifest

  1. build.image_sha256: deterministic image hash of the sealed runtime.
  2. data.nonce: random challenge value bound into the run.
  3. audit_log.merkle_root: anchor for the append-only log of artifacts.
  4. anchors: txids/tags that time-anchor the proof (e.g., IPIC, Git tag).
  5. witnesses: independent attestation letters (PDF hashes).

Manifests are YAML files stored alongside this page.
